Claude Code技能配置与执行机制详解

主要内容：详细讲解 baoyu-skills 的配置方法，包括 .env 文件的创建位置、API Key 配置、EXTEND.md 的作用、配置优先级，以及 Skill 如何检测选择 Provider。

最近在折腾 baoyu-skills 的时候，踩了不少配置相关的坑。这篇文章把我的实践经验和理解整理出来，希望能帮到同样在配置过程中困惑的朋友。

先说结论

baoyu-skills 的配置其实并不复杂，但文档里有些细节说得不够清楚。核心就是两件事：

  1. .env 文件需要手动创建，放在正确的目录
  2. API Key 按需配置，不是所有 skill 都需要

下面展开说说我踩过的坑和理解到的机制。

EXTEND.md 的作用和配置优先级

EXTEND.md 是 baoyu-skills 的一个特色设计，它允许你定义默认配置和行为扩展。

什么是 EXTEND.md？

简单说，EXTEND.md 是一个 YAML 格式的配置文件，可以：

  • 设置默认 Provider
  • 设置默认 Model
  • 设置默认 Quality
  • 设置默认 Aspect Ratio

配置优先级

这是最容易搞混的部分，我用表格说明：

优先级 配置来源 说明
1 (最高) CLI 参数 命令行直接指定的参数
2 EXTEND.md 扩展配置文件
3 环境变量 process.env
4 项目级 .env <cwd>/.baoyu-skills/.env
5 (最低) 用户级 .env ~/.baoyu-skills/.env

实际含义：如果你在 EXTEND.md 里设置了 default_provider: google，同时又配置了多个 API Key，会优先使用 Google。

每个 Skill 都有 loadEnv 吗？

不是所有 skill 都需要 loadEnv。

分类来说：

Skill 如何检测和选择 Provider？

这是我觉得设计得比较巧妙的部分。Skill 会自动检测可用的 Provider。

检测逻辑

优先级顺序：Google → OpenAI → Azure → OpenRouter → DashScope → Z.AI → MiniMax → Replicate → Jimeng → Seedream

这意味着如果你同时配置了多个 API Key，skill 会按这个顺序自动选择。

Bash Tool 调用机制

当你在 Claude Code 中调用一个 skill 时，实际流程是：

展开来说：

  1. Claude 接收你的自然语言请求
  2. 解析出要调用的 skill 名称和参数
  3. 读取 SKILL.md，找到脚本路径和执行方式
  4. 构建并执行 bash 命令
  5. 脚本执行完成后，输出返回给 Claude
  6. Claude 整理结果，呈现给你

总结

折腾完这些配置，我的几点心得：

  • .env 必须手动创建，不会自动生成，记住放在 ~/.baoyu-skills/.env

  • 按需配置，不用一次性把所有 API Key 都配齐。用啥配啥，省得管理麻烦

  • 优先级要清楚：CLI > EXTEND.md > 环境变量 > 项目 .env > 用户 .env。出问题时按这个顺序排查

  • 不是所有 skill 都需要 API Key，本地工具类的 skill 拿来就能用，比如 translate、compress-image、format-markdown

  • 多 Provider 配置时注意优先级，如果想让某个 Provider 生效，要么只配它，要么在 EXTEND.md 里指定 default_provider
