Everything Claude Code (ECC) 深度解析

这篇文章系统解析 everything-claude-code 项目的核心机制，包括 Agent 触发、Hooks 系统、Commands 调用等关键概念，帮助你理解这个 Claude Code 插件库是如何工作的。

一、项目概述

ECC 是什么？

everything-claude-code（简称 ECC）是一个 Claude Code 插件库，提供了大量开箱即用的 AI 编程工具。

组件 数量 作用
Agents 47个 专业子代理（代码审查、架构设计、测试等）
Skills 181个 工作流技能（TDD、安全审查、前端模式等）
Commands 79个 斜杠命令（/tdd、/plan 等）
Hooks 多个 自动化钩子（提交检查、格式化等）
Rules 多个 编码规则（安全、测试、风格等）
MCP配置 14个 外部服务集成

二、CLAUDE.md 的作用

给谁看的？

CLAUDE.md 是给 Claude（AI）读的，不是给人类读的。

文件 给谁看 你需要读吗
README.md 人类 ✅ 需要，了解项目
CLAUDE.md Claude ❌ 不需要，AI 自己读
AGENTS.md Claude ❌ 不需要
CONTRIBUTING.md 人类 ✅ 如果要贡献代码

文件位置与作用范围

位置 作用
项目/CLAUDE.md 项目级指令（只对这个项目生效）
~/.claude/CLAUDE.md 全局指令（所有项目生效）

实际效果

简单说：CLAUDE.md = 给 Claude 的”入职文档”，告诉 Claude 这个项目是什么、怎么干活、遵循什么规则。

三、Agent 触发机制

单 Agent vs 多 Agent

单 Agent 场景：你直接和 Claude 对话，它就是主 agent。

多 Agent 场景：主 Claude 判断需要委派，调用 Agent tool 启动子 agent。

触发条件

Agent 不会自动触发，需要满足以下条件之一：

条件 说明
description 匹配 agent 的 description 字段包含关键词如 “Use PROACTIVELY when…”
AGENTS.md 引导 文件中明确写了 “复杂功能请求 → planner”
用户明确指定 “用 code-reviewer 审查这段代码”
规则强制 rules 中写了 “代码修改后必须使用 code-reviewer”

Agent 文件格式

关键字段：

  • name - agent 名称
  • description - 描述，包含触发条件
  • tools - 可使用的工具
  • model - 使用的模型

四、Hooks 系统

Hooks 是什么？

Hooks 是 事件驱动的自动化脚本，在工具调用时自动触发。

关键点：钩子是通过工具调用触发的，不是通过提示词触发的。

hooks.json 的位置

位置 作用
项目/hooks/hooks.json 项目级钩子
~/.claude/settings.json 全局钩子配置
~/.claude/settings.local.json 本地覆盖配置

钩子类型

类型 时机 能做什么
PreToolUse 工具执行前 可以阻止执行 (exit 2) 或警告
PostToolUse 工具执行后 分析输出，但不能阻止
Stop 每次响应结束后 格式化、通知等
SessionStart 会话开始时 加载上下文
SessionEnd 会话结束时 保存状态、清理

触发流程

用户说什么不重要，重要的是 Claude 调用了什么工具。

退出码含义

退出码 含义
0 成功，继续执行
2 阻止工具调用（仅 PreToolUse）
其他非零 错误，记录但不阻止

实际钩子功能示例

钩子 触发时机 功能
pre:bash:block-no-verify Bash 执行前 阻止 --no-verify 绕过 git hooks
pre:bash:commit-quality 提交前 检查 lint、commit 格式、敏感信息
post:bash:command-log-audit Bash 执行后 记录所有 bash 命令
stop:format-typecheck 响应结束时 批量格式化和类型检查
stop:desktop-notify 响应结束时 发送桌面通知

五、Commands 调用方式

怎么触发？

Commands 必须用户主动输入 /命令名 才触发。

与 Skills、Agents、Hooks 的区别

类型 谁触发 怎么触发
Command 用户 用户输入 /命令名
Skill Claude 或用户 Claude 主动调用，或用户 /skill名
Agent Claude Claude 通过 Agent tool 委派
Hook 系统 工具调用时自动触发
Rule 始终生效 作为 Claude 的行为准则

六、推荐的通用 Agents

按推荐度排序

1. planner（规划专家）⭐⭐⭐

用途：复杂功能实现前，先规划再动手

  • 分析需求
  • 拆分步骤
  • 识别依赖和风险
  • 给出实现顺序

2. code-reviewer（代码审查专家）⭐⭐⭐

用途：写完代码后自动审查

  • 安全检查（SQL注入、XSS、硬编码密钥）
  • 代码质量（大函数、深嵌套、错误处理）
  • React/Next.js 模式检查
  • 性能问题

3. tdd-guide（测试驱动开发专家）⭐⭐⭐

用途：先写测试再写代码

  • Red-Green-Refactor 循环
  • 确保 80%+ 覆盖率
  • 边界情况测试

4. code-explorer（代码探索专家）⭐⭐⭐

用途：深度分析现有代码

  • 发现入口点
  • 追踪执行路径
  • 映射架构层
  • 识别模式和依赖

5. architect（架构设计专家）⭐⭐

用途：系统架构设计

  • 可扩展性设计
  • 技术决策权衡
  • 架构决策记录（ADR）

6. security-reviewer（安全审查专家）⭐⭐

用途：提交前安全检查

  • 漏洞检测
  • 敏感数据暴露
  • 认证授权问题

七、总结

ECC 的核心价值

  1. 专业化 - 47 个专业 agent，每个都有明确的职责和触发条件
  2. 自动化 - Hooks 系统在工具调用时自动执行检查
  3. 规范化 - Rules 系统确保代码质量和安全
  4. 可扩展 - Skills 系统支持自定义工作流

安装建议

使用建议

  1. 常用三件套：planner → tdd-guide → code-reviewer
  2. 理解大项目：code-explorer + code-architect
  3. 架构决策：architect
  4. 安全敏感代码：security-reviewer

核心概念速查

概念 触发方式 给谁用
Agent Agent tool 委派 Claude
Skill Skill tool 或 /skill名 Claude 或用户
Command 用户输入 /命令名 用户
Hook 工具调用时自动触发 系统
Rule 始终生效 Claude
CLAUDE.md Claude 启动时自动读取 Claude

ECC 本质上是一个”增强包”，让你的 Claude Code 变得更强大、更专业。就像给 VS Code 装插件一样，给 Claude Code 装这个插件，就能获得 47 个专业 agent、自动化钩子、规范化工作流等能力。
