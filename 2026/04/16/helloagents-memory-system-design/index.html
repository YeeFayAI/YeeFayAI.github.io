HelloAgents 框架记忆系统设计解析

摘要：记忆是智能体的核心能力之一。HelloAgents 框架借鉴认知科学的记忆模型，设计了四层记忆架构，并整合 RAG 系统实现知识检索。本文深入解析这一设计的技术实现与工程考量。

为何智能体需要记忆

大语言模型存在两个根本性局限：无状态导致的对话遗忘和内置知识的局限性。

对话遗忘问题

LLM 设计上是无状态的，每一次请求都是独立的计算。这意味着：

  • 上下文丢失：长对话中早期信息可能因窗口限制而丢失
  • 个性化缺失：无法记住用户偏好和习惯
  • 学习能力受限：无法从过往经验中学习
  • 一致性问题：多轮对话可能出现前后矛盾

知识局限问题

LLM 的知识完全来自训练数据，存在：

  • 知识时效性：训练数据有时间截止点
  • 专业领域深度：通用模型在特定领域知识不足
  • 事实准确性：可能产生幻觉
  • 可解释性：缺乏信息来源追溯

从认知科学获得启发

人类记忆系统是多层级的认知结构，这为设计智能体记忆提供了理论框架。

人类记忆的层次结构

记忆类型 持续时间 容量 功能
感觉记忆 0.5-3 秒 巨大 暂存感官信息
工作记忆 15-30 秒 7±2 项目 当前任务处理
长期记忆 可达终生 几乎无限 知识与经验存储

长期记忆进一步分为程序性记忆（技能习惯）和陈述性记忆（可用语言表达的知识），后者又包含语义记忆（一般知识概念）和情景记忆（个人经历事件）。

HelloAgents 四层记忆架构

借鉴认知科学，框架设计了四层架构：

基础设施层

提供记忆系统的底层支撑：

  • MemoryManager：统一调度和协调的中心控制器
  • MemoryItem：标准化的记忆数据结构
  • MemoryConfig：系统参数配置管理
  • BaseMemory：通用接口定义

记忆类型层

实现不同特性的记忆类型：

类型 特性 存储特点 典型用途
WorkingMemory 临时信息 纯内存，TTL 管理 当前对话上下文
EpisodicMemory 具体事件 SQLite + Qdrant 历史交互记录
SemanticMemory 抽象知识 Qdrant + Neo4j 知识图谱关系
PerceptualMemory 多模态数据 SQLite + Qdrant 图像、音频等

存储后端层

提供多样化的持久化方案：

  • QdrantVectorStore：高性能向量检索，支持语义相似度搜索
  • Neo4jGraphStore：知识图谱管理，支持实体关系查询
  • SQLiteDocumentStore：结构化持久化，轻量级本地存储

嵌入服务层

提供文本向量化能力：

  • DashScopeEmbedding：阿里云通义千问嵌入，云端 API
  • LocalTransformerEmbedding：本地 sentence-transformers，离线部署
  • TFIDFEmbedding：轻量级 TF-IDF，作为兜底方案

记忆系统工作流程

记忆的形成经历五个认知阶段，每个阶段映射为具体的技术操作：

编码（Encoding）

将感知到的信息转换为可存储的形式。在技术实现中，这涉及文本预处理、实体抽取、向量化等操作。

存储（Storage）

将编码后的信息保存在记忆系统中。不同类型的记忆选择不同的存储后端：工作记忆存内存，情景记忆存 SQLite，语义记忆存图数据库。

检索（Retrieval）

根据需要从记忆中提取相关信息。支持多种检索策略：向量相似度、关键词匹配、时间范围、实体关联等。

整合（Consolidation）

将短期记忆转化为长期记忆。框架实现了记忆的”升级”机制：工作记忆中频繁访问的信息可自动提升为情景记忆或语义记忆。

遗忘（Forgetting）

删除不重要或过时的信息。实现了 TTL 过期机制、容量限制策略、重要性评分淘汰等机制。

RAG 系统设计

RAG（检索增强生成）专注于外部知识的获取和利用，与记忆系统形成互补。

架构设计

与记忆系统的关系

记忆系统和 RAG 系统被设计为两个独立的工具：

  • memory_tool：存储和维护对话过程中的交互信息
  • rag_tool：从知识库中检索相关信息作为上下文

两者可以协同工作：RAG 检索到的重要信息可自动存储到记忆系统，形成知识的积累。

工程实现考量

存储后端选择

场景 推荐方案 理由
开发测试 SQLite + 内存 零配置，快速启动
生产环境 Qdrant + Neo4j 高性能，可扩展
知识密集型 Qdrant + Neo4j 支持复杂关系查询
资源受限 SQLite + TF-IDF 轻量级，低资源占用

嵌入模型选择

云端 API（如 DashScope）适合追求效果和便捷性的场景；本地模型适合对数据隐私有要求或需要离线部署的场景；TF-IDF 作为兜底方案，在无法使用深度模型时提供基础能力。

性能优化策略

  • 批量处理：文档分块和向量化采用批量操作
  • 缓存机制：频繁访问的记忆项缓存到工作记忆
  • 异步索引：新记忆异步建立向量索引，不阻塞主流程
  • 命名空间隔离：不同用户/会话使用独立命名空间

总结

  • 智能体记忆系统借鉴认知科学的多层级设计，实现不同特性的记忆类型
  • 四层架构（基础设施、记忆类型、存储后端、嵌入服务）实现了关注点分离
  • 多种存储后端和嵌入服务提供了灵活的技术选型空间
  • RAG 系统与记忆系统互补，解决外部知识获取问题
  • 工程实现需在效果、性能、资源占用之间权衡
